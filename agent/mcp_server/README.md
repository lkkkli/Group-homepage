# 天气出行智能助手 MCP 服务器

这是一个基于 Model Context Protocol (MCP) 的智能助手服务器，提供天气查询和自动出行计划生成功能。

## 功能特性

- 天气查询: 查询指定地点的实时天气信息
- 出行计划生成: 基于天气情况自动生成出行建议
- 多语言支持: 支持中英文地点名称和天气描述
- 智能建议: 基于天气类型提供针对性出行建议

---

## 需求描述

### 项目目标
创建一个智能化的天气出行助手，能够为用户提供实时的天气信息查询和基于天气情况的出行计划建议，帮助用户做出更明智的出行决策。

### 目标用户
- 准备出行的旅行者
- 需要了解天气情况的户外活动爱好者
- 关心天气对日常活动影响的用户
- 开发者（通过API集成）

### 核心功能需求
1. 实时天气查询
   - 支持全球主要城市天气查询
   - 提供温度、湿度、风速等详细气象数据
   - 支持中英文城市名称输入

2. 智能出行建议
   - 根据天气情况自动生成出行建议
   - 区分晴天、雨天、雾天等不同天气类型
   - 提供针对性的装备和活动建议

---

## 业务流描述

### 主要业务流程

#### 天气查询流程
用户输入 → 城市名称解析 → API调用 → 数据处理 → 结果展示

#### 出行计划生成流程
天气查询 → 天气类型识别 → 规则匹配 → 建议生成 → 结果输出

#### 完整用户交互流程
用户启动 → 显示欢迎信息 → 等待输入 → 查询天气 → 询问计划需求 → 生成建议 → 显示结果 → 继续/退出

### 关键业务规则

#### 天气类型识别规则
- 晴天: 包含 "clear", "sunny", "fair", "晴" 等关键词
- 雨天: 包含 "rain", "shower", "drizzle", "雨", "雪" 等关键词
- 云天: 包含 "cloud", "overcast", "partly", "云" 等关键词
- 雾天: 包含 "fog", "mist", "haze", "雾" 等关键词

#### 出行建议生成规则
- 装备建议: 根据天气类型推荐雨具、防晒用品等
- 活动建议: 区分室内/室外活动优先级
- 安全提醒: 特殊天气条件下的注意事项
- 通用提示: 交通、证件等基本出行提醒

### 数据流图
用户输入 → 输入验证 → 天气API调用 → 数据解析 → 建议生成 → 结果格式化 → 用户界面展示

## 实现说明

### 系统架构

用户界面层    业务逻辑层    数据访问层
- 命令行界面   - 天气查询逻辑  - wttr.in API
- Web界面      - 建议生成引擎  - 数据缓存
- MCP接口      - 输入验证      - 错误处理

### 工作流程

#### 服务器启动流程
1. 加载配置
2. 初始化MCP服务器
3. 注册工具函数
4. 启动stdio监听
5. 等待客户端连接

#### 工具调用流程
1. 接收MCP客户端请求
2. 解析工具名称和参数
3. 执行相应业务逻辑
4. 格式化返回结果
5. 通过MCP协议返回

### 接口设计

#### MCP工具接口
```json
{
  "get_weather": {
    "input": {"location": "string"},
    "output": "天气信息字符串"
  },
  "generate_travel_plan": {
    "input": {
      "destination": "string",
      "weather_info": "string"
    },
    "output": "出行建议字符串"
  }
}
```
